Introduction

The OTA Community Edition is open-source server software to allow over-the-air (OTA) updates of compatible clients . It is comprised of a number of services which together make up the OTA system.

This documentation is to help you try out the ota-community-edition .

Requirements

You will need the following to try this out

  1. linux system( preferrably ubuntu)
  2. Aktualizr

Step 1. Generate Device

Download the script gen-device.sh and run it to generate a virtual device, the script will generate a new device with its universally unique identifier [uuid] and configurations in ota-ce-gen/devices/[uuid]/

  • #In the directory you choose:
  • $ wget https://uptanedemo.org/gen-device.sh
  • $ chmod +x gen-device.sh
  • $ ./gen-device.sh
  • #try connecting the aktualizr
  • $ cd ota-ce-gen/devices/[uuid]
  • $ aktualizr --run-mode=once --config=config.toml

We need the two directories that the above script will make on being executed the very first instance: certs/ and ota-ce-gen/ for the next step

Download gen-device.sh

Step 2. Generate Credentials

Download the script get-credentials.sh in the same directory where you downloaded gen-device.sh to create credentials

  • #In the same directory as step 1:
  • $ wget https://uptanedemo.org/get-credentials.sh
  • $ chmod +x get-credentials.sh
  • $ ./get-credentials.sh https uptanedemo.org

We need the zip file that the above script will make on being executed : ota-ce-gen/credentials.zip for the next step

Download get-credentials.sh

Step 3. Install OTA-CLI

Install the ota-cli tool to to interact with uptanedemo

  • #In the directory you choose:
  • $ git clone https://github.com/simao/ota-cli.git
  • $ cd ota-cli
  • #run the following and follow the onscreen instructions
  • $ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
  • #Build the tool
  • $ make ota

Step 4. Initialize OTA-CLI

Initiate the ota-cli tool to to interact with uptanedemo's services

  • $ ota init \
  • --credentials /path/to/ota-ce-gen/credentials.zip --campaigner https://campaigner.uptanedemo.org --director https://director.uptanedemo.org --registry https://deviceregistry.uptanedemo.org

Create target.toml

Next we create the target file, target.toml based on the o/p above. You need to change the fields length and hash according to the above output



		[ota-ce-device]
		target_format = "binary"
		generate_diff = true

		# required metadata specifying update target
		[ota-ce-device.to]
		name = "mypkg"
		version = "0.0.1"
		length = 830
		hash = "70b22c2b8c857d1416a16e04cbfdf2b4d00d8b84000cc809af2061301ccfecb1"
		method = "sha256"

Remember that output of this command is an uuid important for sending update

Create Update

Create the update using ota

  • # you need to save or copy the o/p of this command
  • $ ota update create -t /path/to/target.toml

Remember that output of this command is an uuid important for sending update

Step 6: Serve the Update

Serving the update

--update [uuid recieved in step 5] --device [uuid of device generated in step 1]

  • $ ota update launch --update uuid --device uuid

The o/p of this command on success must be: OK

Step 7: Update the device

Now we try to update the device we made in step 1 in this step. Go inside the device directory

The uuid in this step must be the same as the uuid of the device you created in step 1 and the device we served the update to in step 6

  • $ cd ota-ce-gen/devices/[uuid]
  • $ aktualizr --run-mode=once --config=config.toml
  • # you should find the update in the following path
  • $ cd ota-ce-gen/devices/[uuid]/storage/images/
